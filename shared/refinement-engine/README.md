# @celebratebanner/refinement-engine

Natural-language refinement of an **already-approved** WOW concept. The customer
picks a concept in the Premium Reveal, then asks for a change in plain words —
"make it more luxurious", "bigger hero", "warmer colors" — and this engine applies
it *within the rules*, recomputes the WOW Score, and re-gates at 90.

```
… WOW Engine → (customer picks a concept) → Refinement Engine → re-gated concept
```

It makes **no creative decision it cannot justify** and produces **no pixels**.

## Public API

```ts
refineConcept(
  selectedConcept,     // WowConcept the customer picked
  customerInstruction, // natural-language request
  creativeBrief,       // the concept's brief (decorative allowlist, etc.)
  memoryProfile,       // read-only context
  options?             // { maxIntents = 3, maxHeroDominance = 0.8 }
): RefinedConcept
```

`RefinedConcept` carries: `accepted`, the parsed `intents`, `appliedRefinements`
(with per-field `changes`), the resulting `concept` (refined when accepted, the
**original untouched** when rejected), `previousWowScore` / `wowScore` /
`attemptedScore`, the full `scoreBreakdown`, `masterpiecePassed`, `rejectionReasons`,
a human `changeLog`, and the `constitution` check results.

## Supported refinement intents (14)

`luxury` · `elegance` · `modern` · `classic` · `minimal` · `celebration` ·
`hero-emphasis` · `typography` · `color` · `decoration` · `lighting` · `emotion` ·
`background` · `energy`

Each intent maps to bounded score deltas and deterministic recipe edits (layout,
color, typography guidance). Intents are detected by word-boundary keyword matching,
returned in a canonical order for stable output.

## The Constitution (never violated)

- **WOW Score ≥ 90** — every refinement recalculates the score; below 90 → **rejected** with an explanation, and the original concept is kept.
- **Never reduce hero dominance** — `hero-emphasis` raises it (bounded by `maxHeroDominance`); a request to shrink the hero is refused.
- **Never change customer names**, **never fabricate people / invent facts**, **never reorder memories without justification** — these requests are refused up front.
- **Never produce clutter** — stacking ornament (decoration + celebration) or decorating with no brief allowlist is refused; more than `maxIntents` at once is refused as over-refinement.
- **Conflicting directions** (e.g. modern + classic) are refused.

A rejected refinement returns the **original concept unchanged** — nothing sub-90 or
constitution-violating ever ships.

## Determinism

No `Date`, `Math.random`, or I/O. Same inputs → byte-identical `RefinedConcept`. The
committed `fixtures/*.json` are snapshots regenerated by `npm run fixtures`.
